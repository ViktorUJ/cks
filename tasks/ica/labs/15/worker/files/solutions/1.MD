# Lab 15 - Reference Solution

Istio version used: `1.29.1` (istioctl is pre-installed on the worker PC).

## Goal

Install Istio with a **customized** configuration using an `IstioOperator` manifest:
- start from the `default` profile;
- set two `meshConfig` options:
  - `accessLogFile: /dev/stdout` (Envoy access logs to stdout),
  - `outboundTrafficPolicy.mode: REGISTRY_ONLY` (block egress to unknown hosts);
- add a **second** ingress gateway component `ingressgateway-internal` alongside the
  default `istio-ingressgateway`.

## Step 1. Write the IstioOperator manifest

```bash
cat > custom-istio.yaml <<'EOF'
apiVersion: install.istio.io/v1alpha1
kind: IstioOperator
metadata:
  name: custom-istio
spec:
  profile: default
  meshConfig:
    accessLogFile: /dev/stdout
    outboundTrafficPolicy:
      mode: REGISTRY_ONLY
  components:
    ingressGateways:
      - name: istio-ingressgateway
        enabled: true
      - name: ingressgateway-internal
        enabled: true
        label:
          istio: ingressgateway-internal
        k8s:
          service:
            type: ClusterIP
EOF
```

## Step 2. Install Istio with the custom manifest

```bash
istioctl install -f custom-istio.yaml -y
```

## Step 3. Verify the installation

```bash
# control plane
kubectl get pods -n istio-system

# both ingress gateways
kubectl get deploy -n istio-system | grep -E 'ingressgateway'

# meshConfig landed in the istio configmap
kubectl get configmap istio -n istio-system -o jsonpath='{.data.mesh}' | grep -E 'accessLogFile|outboundTrafficPolicy|REGISTRY_ONLY'
```

Expected:
- `istiod` is `Running`;
- deployments `istio-ingressgateway` and `ingressgateway-internal` are both ready;
- the `istio` configmap contains `accessLogFile: /dev/stdout` and
  `outboundTrafficPolicy.mode: REGISTRY_ONLY`.

## (optional) Step 4. See REGISTRY_ONLY in action

```bash
kubectl create namespace demo
kubectl label namespace demo istio-injection=enabled
kubectl -n demo run curl --image=curlimages/curl -it --rm --restart=Never -- \
  sh -c 'curl -sS -o /dev/null -w "%{http_code}\n" http://httpbin.org/get'
# -> 502 : egress is blocked because the host is not in the mesh registry
```

With `outboundTrafficPolicy: REGISTRY_ONLY`, calls to external hosts that are not
declared via a `ServiceEntry` are rejected. To allow such traffic you would add a
`ServiceEntry` (see Lab 05).

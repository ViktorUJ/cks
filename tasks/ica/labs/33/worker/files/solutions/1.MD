# Lab 33 - Reference Solution

Istio `1.29.1` (default profile, cluster-wide discovery, no Sidecar scope) and OPA
Gatekeeper are pre-installed. Three namespaces exist:

- `app` (injected, `mesh=enabled`) - `frontend`;
- `shop` (injected, `mesh=enabled`) - `catalog` + a sidecar-less `probe`;
- `legacy` (NOT injected, no `mesh` label) - `legacy-app`.

`istioctl` is available on the worker PC.

## Goal

1. Enable **discovery selectors** so istiod only tracks `mesh=enabled` namespaces
   (drops `legacy`).
2. Add a **Sidecar** egress scope in `app` so its proxies only know `app` + `istio-system`
   (drops `shop`).
3. Observe istiod **golden signals**.
4. Enforce a deploy policy with **OPA Gatekeeper**.

## Step 1. Discovery selectors

Re-apply the install with `meshConfig.discoverySelectors` matching `mesh=enabled`
(`istio-system`, `app`, `shop` carry it; `legacy` does not):

```bash
cat <<EOF > /tmp/iop.yaml
apiVersion: install.istio.io/v1alpha1
kind: IstioOperator
spec:
  profile: default
  meshConfig:
    discoverySelectors:
      - matchLabels:
          mesh: enabled
EOF
istioctl install -f /tmp/iop.yaml -y
```

Verify `legacy` disappears from the mesh (from a proxy that has no Sidecar scope yet):

```bash
istioctl proxy-config clusters deploy/catalog.shop | grep legacy-app || echo "legacy dropped"
```

## Step 2. Sidecar egress scope in app

```bash
kubectl apply -f - <<'EOF'
apiVersion: networking.istio.io/v1
kind: Sidecar
metadata:
  name: default
  namespace: app
spec:
  egress:
    - hosts:
        - "./*"
        - "istio-system/*"
EOF
```

`./*` = the proxy's own namespace (`app`), plus `istio-system/*` for control-plane and
gateways. Everything else (including `shop`) is removed from `app` proxies:

```bash
istioctl proxy-config clusters deploy/frontend.app | grep catalog.shop || echo "shop dropped from app"
```

## Step 3. istiod golden signals

```bash
# convergence time + push count (the key control-plane signals)
kubectl exec -n shop deploy/probe -c probe -- \
  curl -s http://istiod.istio-system:15014/metrics \
  | grep -E 'pilot_proxy_convergence_time|pilot_xds_pushes'

# how many proxies are connected / config sync state
istioctl proxy-status
```

`pilot_proxy_convergence_time` is the main signal: how long a change takes to reach the
proxies. `pilot_xds_pushes` counts pushes. Rising convergence time or push errors means
the control plane is under pressure - the scope levers in steps 1-2 are what reduce it.

## Step 4. OPA Gatekeeper deploy policy

Require every namespace to declare the injection label (a classic deploy-hygiene policy
from the chapter). Create the template and the constraint:

```bash
kubectl apply -f - <<'EOF'
apiVersion: templates.gatekeeper.sh/v1
kind: ConstraintTemplate
metadata:
  name: k8srequiredlabels
spec:
  crd:
    spec:
      names:
        kind: K8sRequiredLabels
      validation:
        openAPIV3Schema:
          type: object
          properties:
            labels:
              type: array
              items:
                type: string
  targets:
    - target: admission.k8s.gatekeeper.sh
      rego: |
        package k8srequiredlabels
        violation[{"msg": msg}] {
          provided := {label | input.review.object.metadata.labels[label]}
          required := {label | label := input.parameters.labels[_]}
          missing := required - provided
          count(missing) > 0
          msg := sprintf("namespace is missing required labels: %v", [missing])
        }
EOF

kubectl apply -f - <<'EOF'
apiVersion: constraints.gatekeeper.sh/v1beta1
kind: K8sRequiredLabels
metadata:
  name: ns-must-have-injection
spec:
  match:
    kinds:
      - apiGroups: [""]
        kinds: ["Namespace"]
  parameters:
    labels: ["istio-injection"]
EOF
```

Verify the policy is enforced (should be **denied**):

```bash
kubectl create ns test-no-label
# Error ... admission webhook "validation.gatekeeper.sh" denied the request:
# namespace is missing required labels: {"istio-injection"}
```

## Why this works

- **Discovery selectors** limit which namespaces istiod watches at all. Namespaces without
  the selector label are invisible to the control plane - their services never become
  clusters/endpoints on any proxy, cutting config size and push work. Biggest win when
  part of the cluster is not in the mesh.
- **Sidecar egress scope** limits, per namespace, which services a proxy is told about.
  With `./*` + `istio-system/*`, an `app` proxy no longer carries config for `shop` (or the
  rest of the mesh). This shrinks each proxy's config and the push fan-out from istiod.
- **Golden signals** (`pilot_proxy_convergence_time`, `pilot_xds_pushes`, connected proxies,
  istiod CPU/mem) are how you see whether the control plane keeps up; the scope levers are
  the main tools to bring convergence time down.
- **OPA Gatekeeper** turns best practices into admission rules: non-compliant resources are
  rejected at creation, not merely discouraged.

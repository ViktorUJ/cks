# Lab 06 — Reference Solution

## Step 1. Enable sidecar injection

```bash
kubectl label namespace default istio-injection=enabled --overwrite
```

## Step 2. Deploy the application

```bash
kubectl apply -f https://raw.githubusercontent.com/ViktorUJ/cks/refs/heads/AG-148/tasks/ica/labs/06/k8s-1/scripts/1.yaml
kubectl rollout restart deployment -n default
kubectl rollout status deployment/ping-pong-v1 -n default
kubectl rollout status deployment/ping-pong-v2 -n default
```

## Step 3. Gateway + VirtualService (route to v1)

```yaml
# gateway.yaml
apiVersion: networking.istio.io/v1
kind: Gateway
metadata:
  name: main-gateway
  namespace: default
spec:
  selector:
    istio: ingressgateway
  servers:
  - port:
      number: 80
      name: http
      protocol: HTTP
    hosts:
    - "myapp.local"
---
apiVersion: networking.istio.io/v1
kind: VirtualService
metadata:
  name: ping-pong-vs
  namespace: default
spec:
  hosts:
  - "myapp.local"
  - "ping-pong"
  gateways:
  - main-gateway
  - mesh
  http:
  - route:
    - destination:
        host: ping-pong
        subset: v1
```

```bash
kubectl apply -f gateway.yaml
```

## Step 4. Load Balancing (DestinationRule)

```yaml
# destination-rule.yaml
apiVersion: networking.istio.io/v1
kind: DestinationRule
metadata:
  name: ping-pong-dr
  namespace: default
spec:
  host: ping-pong
  trafficPolicy:
    loadBalancer:
      simple: ROUND_ROBIN          # global algorithm
    portLevelSettings:
    - port:
        number: 8080
      loadBalancer:
        simple: LEAST_REQUEST      # per-port override for 8080
  subsets:
  - name: v1
    labels:
      version: v1
  - name: v2
    labels:
      version: v2
```

```bash
kubectl apply -f destination-rule.yaml
```

Observe the load balancing across the three v1 replicas:

```bash
for i in $(seq 12); do curl -s http://myapp.local:32080 | grep 'Server Name'; done | sort | uniq -c
```

Supported `simple` algorithms: `ROUND_ROBIN`, `LEAST_REQUEST`, `RANDOM`, `PASSTHROUGH`.

## Step 5. Traffic Mirroring (shadow traffic to v2)

```yaml
# mirror-vs.yaml
apiVersion: networking.istio.io/v1
kind: VirtualService
metadata:
  name: ping-pong-vs
  namespace: default
spec:
  hosts:
  - "myapp.local"
  - "ping-pong"
  gateways:
  - main-gateway
  - mesh
  http:
  - route:
    - destination:
        host: ping-pong
        subset: v1          # 100% of real responses come from v1
    mirror:
      host: ping-pong
      subset: v2            # a copy of every request is sent to v2
    mirrorPercentage:
      value: 100.0
```

```bash
kubectl apply -f mirror-vs.yaml
```

## Verify

```bash
# client always gets v1
for i in $(seq 10); do curl -s http://myapp.local:32080 | grep 'Server Name'; done

# v2 receives mirrored requests (fire-and-forget, response is ignored)
kubectl exec -n default deploy/ping-pong-v2 -c istio-proxy -- \
  pilot-agent request GET stats | grep -E 'inbound\|8080.*rq_total'
```

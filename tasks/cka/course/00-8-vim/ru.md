# Глава 0.8. vim за 15 минут: выжить и настроить под YAML

> **Для кого эта глава.** Короткий практический минимум по редактору `vim`. На экзаменах
> CKA/CKAD и на нодах кластера это редактор по умолчанию, и правит им вы будете постоянно
> (манифесты, конфиги, `/etc/...`). Цель - не «выучить vim», а не терять на нём время:
> зайти, отредактировать, сохранить, выйти и правильно настроить отступы под YAML. Если
> вы уже уверенно работаете в vim - пропускайте главу.

## 0.8.1. Два режима - главное, что нужно понять

Вся путаница новичка в vim - из-за **режимов**. Клавиши делают разное в зависимости от
того, в каком вы режиме.

```mermaid
flowchart LR
    normal["NORMAL<br>команды: перемещение,<br>удаление, сохранение"] -->|"нажать i"| insert["INSERT<br>обычный набор текста"]
    insert -->|"нажать Esc"| normal
    style normal fill:#326ce5,color:#fff
    style insert fill:#0f9d58,color:#fff
```

- **NORMAL** (по умолчанию при входе) - клавиши это команды, а не текст.
- **INSERT** - обычный ввод текста; входят клавишей `i`, выходят - `Esc`.

Правило выживания: **если что-то пошло не так - жми `Esc`** (вернёшься в Normal), а потом
уже команду.

## 0.8.2. Минимум команд, чтобы выжить

Этого набора хватает на 95% работы на экзамене:

| Действие | Клавиши (из режима Normal) |
|----------|----------------------------|
| Войти в редактирование | `i` |
| Выйти из редактирования | `Esc` |
| Сохранить | `:w` + Enter |
| Сохранить и выйти | `:wq` или `ZZ` |
| Выйти без сохранения | `:q!` |
| Отменить / вернуть | `u` / `Ctrl+r` |
| Удалить строку | `dd` |
| Скопировать / вставить строку | `yy` / `p` |
| В начало / конец файла | `gg` / `G` |
| Поиск | `/текст` + Enter (следующее - `n`) |
| Номера строк | `:set number` |

Открыть файл: `vim file.yaml`. Всё. Больше на экзамене обычно и не нужно.

## 0.8.3. Настройка под YAML - обязательно

Главная беда в vim при работе с Kubernetes: **табы вместо пробелов** и «съезжающие»
отступы. YAML запрещает табы (глава 0.6), а vim по умолчанию может вставлять их. Поэтому
первое, что делают на экзамене, - создают `~/.vimrc`:

```vim
set expandtab       " Tab вставляет пробелы, а не таб
set tabstop=2       " ширина таба - 2 пробела
set shiftwidth=2    " отступ автоиндента - 2 пробела
set number          " номера строк
set autoindent      " сохранять отступ на новой строке
syntax on           " подсветка синтаксиса
```

С этим конфигом клавиша Tab даёт два пробела, и манифесты не ломаются на отступах.
Настройку `~/.vimrc` стоит сделать **первым делом** на экзамене (это входит в стартовые
действия из главы 1).

## 0.8.4. Ловушка при вставке (paste)

Когда вы вставляете (мышью) готовый YAML в vim в режиме Insert с включённым
`autoindent`, отступы **каскадно нарастают** - каждая строка сдвигается всё правее.
Лечится так:

```vim
:set paste      " перед вставкой - отключает автоотступ
" ... вставляете текст ...
:set nopaste    " после вставки - вернуть обычный режим
```

Если увидели «лесенку» из отступов после вставки - это оно. `:set paste`, отменить (`u`),
вставить заново.

## 0.8.5. Мини-глоссарий

- **Режим Normal** - командный режим vim (по умолчанию); клавиши это команды.
- **Режим Insert** - режим набора текста; вход `i`, выход `Esc`.
- **`:wq` / `:q!`** - сохранить и выйти / выйти без сохранения.
- **`~/.vimrc`** - файл настроек vim (отступы, номера строк).
- **`expandtab`** - заменять таб пробелами (критично для YAML).
- **`:set paste`** - режим вставки без автоотступа (против «лесенки»).

## 0.8.6. Итоги главы

- В vim два режима: Normal (команды) и Insert (текст); переход - `i` и `Esc`. Растерялись
  - жмите `Esc`.
- Выживательный минимум: `i`, `Esc`, `:wq`, `:q!`, `u`, `dd`, `/поиск`, `:set number`.
- Под YAML обязательно настроить `~/.vimrc` (`expandtab`, `tabstop=2`, `shiftwidth=2`) -
  иначе табы сломают манифесты.
- При вставке готового текста используйте `:set paste`, чтобы отступы не «съезжали».

## 0.8.7. Как это пригодится: на экзамене и в реальной работе

**На экзамене.** Редактор - ваш основной инструмент 2 часа подряд. Потерянные на возню с
vim минуты - это нерешённые задачи. Настроенный `~/.vimrc` и десяток команд экономят
время на каждой задаче с манифестом.

**В реальной работе.** vim есть на любой Linux-ноде без установки - когда правишь конфиг
на сервере по SSH, выбора часто нет. Базовое владение vim - обязательный навык инженера.

## Задание 1. Под `multi-pod` с двумя контейнерами

```yaml
apiVersion: v1
kind: Pod
metadata: {name: multi-pod}
spec:
  containers:
  - name: alpha
    image: nginx
    env: [{name: name, value: alpha}]
  - name: beta
    image: busybox
    command: ["sleep", "4800"]
    env: [{name: name, value: beta}]
```

## Задание 2. Sidecar-логгер: общий emptyDir, второй контейнер собирает логи

```yaml
apiVersion: v1
kind: Pod
metadata: {name: logger}
spec:
  volumes:
  - name: logs
    emptyDir: {sizeLimit: 500Mi}
  containers:
  - name: app
    image: busybox
    command: ["/bin/sh","-c","while true; do echo hi >> /log/app.log; sleep 5; done"]
    volumeMounts: [{name: logs, mountPath: /log}]
  - name: log
    image: busybox
    command: ["/bin/sh","-c","tail -F /log/app.log"]
    volumeMounts: [{name: logs, mountPath: /log}]
```

`log`-контейнер читает из общего тома то, что пишет `app`, и выводит в stdout —
классический паттерн sidecar-сборщика логов.

## Задание 3. Под `redis-storage` с emptyDir

```yaml
apiVersion: v1
kind: Pod
metadata: {name: redis-storage}
spec:
  containers:
  - name: redis-storage
    image: redis:alpine
    volumeMounts:
    - {name: data, mountPath: /data/redis}
  volumes:
  - name: data
    emptyDir: {sizeLimit: 500Mi}
```

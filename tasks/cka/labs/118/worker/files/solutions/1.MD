# Лаба 118 — эталонное решение (сертификаты, CoreDNS, сеть кластера)

Отчёты сохраняем на рабочей машине в каталог `/home/ubuntu/answers/`. К control plane
подключаемся по SSH: `ssh k8s1_controlPlane_1`.

## Задание 1. Health-check сертификатов → `certs.txt`

Сертификаты кластера лежат на control plane в `/etc/kubernetes/pki/`. Читаем их
`openssl`-ом ([официальная документация: Certificate Management with kubeadm](https://kubernetes.io/docs/tasks/administer-cluster/kubeadm/kubeadm-certs/)).

```bash
ssh k8s1_controlPlane_1

# срок действия сертификата API-сервера
sudo openssl x509 -in /etc/kubernetes/pki/apiserver.crt -noout -enddate
# notAfter=Jul 20 10:00:00 2026 GMT   <- значение после "notAfter="

# CN центра сертификации кластера
sudo openssl x509 -in /etc/kubernetes/pki/ca.crt -noout -subject
# subject=CN = kubernetes   <- CN = kubernetes

# удобный общий обзор сроков:
sudo kubeadm certs check-expiration
```

На рабочей машине запишите отчёт (значение `notAfter` — ровно строкой после знака `=`):

```bash
mkdir -p /home/ubuntu/answers
cat > /home/ubuntu/answers/certs.txt <<EOF
apiserver_notafter=Jul 20 10:00:00 2026 GMT
ca_cn=kubernetes
EOF
```

> Подставьте своё значение `notAfter` из вывода `openssl`.

## Задание 2. Починить CoreDNS

CoreDNS — это Deployment в `kube-system`
([официальная документация: DNS for Services and Pods](https://kubernetes.io/docs/concepts/services-networking/dns-pod-service/),
[Debugging DNS Resolution](https://kubernetes.io/docs/tasks/administer-cluster/dns-debugging-resolution/)).

```bash
kubectl -n kube-system get deploy coredns          # READY 0/0 — реплики обнулены
kubectl -n kube-system scale deployment coredns --replicas=2
kubectl -n kube-system rollout status deploy coredns

# проверка резолвинга:
kubectl run dnstest --image=busybox:1.36 --restart=Never -it --rm -- \
  nslookup kubernetes.default
```

## Задание 3. Факты о сети кластера → `net.txt`

```bash
# Pod CIDR кластера (--cluster-cidr у kube-controller-manager)
ssh k8s1_controlPlane_1 \
  "sudo grep -o 'cluster-cidr=[^ ]*' /etc/kubernetes/manifests/kube-controller-manager.yaml"
# cluster-cidr=10.0.0.0/16

# CNI: DaemonSet агента сети в kube-system
kubectl -n kube-system get ds | grep calico
# calico-node ...
```

Документация: [Cluster Networking](https://kubernetes.io/docs/concepts/cluster-administration/networking/),
[kubeadm: pod-network-cidr](https://kubernetes.io/docs/reference/setup-tools/kubeadm/kubeadm-init/).

На рабочей машине запишите отчёт:

```bash
cat > /home/ubuntu/answers/net.txt <<EOF
pod_cidr=10.0.0.0/16
cni_daemonset=calico-node
EOF
```

## Задание 4. Понизить ndots для пода (dnsConfig)

kubelet прописывает подам `/etc/resolv.conf` с `options ndots:5`: имена, где меньше 5
точек, сначала перебираются с search-суффиксами. Для внешних имён (`api.github.com` — 2
точки) это лишние NXDOMAIN-запросы. Понизить порог для пода можно через `dnsConfig`
([официальная документация: DNS for Services and Pods — Pod DNS Config](https://kubernetes.io/docs/concepts/services-networking/dns-pod-service/#pod-dns-config)).

Сначала под с дефолтным DNS:

```bash
kubectl run dns-default --image=viktoruj/ping_pong:alpine --restart=Never

# посмотреть его resolv.conf: там будет "options ndots:5"
kubectl exec dns-default -- cat /etc/resolv.conf
```

Затем под с пониженным `ndots` (декларативно, т.к. `--dry-run` не умеет dnsConfig):

```yaml
# dns-tuned.yaml
apiVersion: v1
kind: Pod
metadata: {name: dns-tuned}
spec:
  dnsConfig:
    options:
    - {name: ndots, value: "2"}
  containers:
  - {name: c, image: viktoruj/ping_pong:alpine}
```

```bash
kubectl apply -f dns-tuned.yaml
kubectl get po dns-tuned -o jsonpath='{.spec.dnsConfig.options}'; echo
```

> Компромисс: `ndots:2` ускоряет внешние имена, но обращение к сервису в **другом**
> namespace по короткому `backend.prod` (2 точки → уже абсолютное) перестанет достраиваться
> через search. Для внешних имён надёжнее FQDN с точкой в конце: `api.github.com.`.

## Задание 5. Отчёт по ndots → `dns.txt`

```bash
# значение ndots из дефолтного пода
kubectl exec dns-default -- cat /etc/resolv.conf | grep -oE 'ndots:[0-9]+'
# ndots:5

cat > /home/ubuntu/answers/dns.txt <<EOF
default_ndots=5
tuned_ndots=2
EOF
```

# Лаба 114 — эталонное решение (troubleshooting)

Маршрут диагностики один: `get` → `describe` (Events) → `logs` → починка.

## Задание 1. ReplicaSet rs-app2223 (ImagePullBackOff)

```bash
kubectl get po -n rsapp                 # STATUS: ImagePullBackOff
kubectl describe po -n rsapp <pod>      # Events: не тянется образ nginx:doesnotexist123
# чиним образ в шаблоне RS
kubectl edit rs rs-app2223 -n rsapp     # image: nginx:doesnotexist123 -> nginx
# RS не пересоздаёт поды сам при изменении шаблона — удалим старые поды
kubectl delete po -n rsapp -l app=rsapp
kubectl get rs rs-app2223 -n rsapp      # READY 2
```

## Задание 2. Service svc-broken (пустые Endpoints)

```bash
kubectl get endpoints svc-broken -n tsvc   # пусто
kubectl describe svc svc-broken -n tsvc    # selector app=WRONG
kubectl get pods -n tsvc --show-labels     # у подов метка app=web
# чиним селектор сервиса
kubectl patch svc svc-broken -n tsvc -p '{"spec":{"selector":{"app":"web"}}}'
kubectl get endpoints svc-broken -n tsvc   # появились адреса
```

## Задание 3. Deployment capp (CreateContainerConfigError)

```bash
kubectl get po -n cfgapp                  # CreateContainerConfigError
kubectl describe po -n cfgapp <pod>       # не найден configMap missing-config
# создаём недостающий ConfigMap
kubectl create configmap missing-config -n cfgapp --from-literal=KEY=value
kubectl rollout status deploy capp -n cfgapp
```

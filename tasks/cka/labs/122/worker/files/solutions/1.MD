# Лаба 122 — эталонное решение (scheduling-дриллы)

Неймспейс `sched` создан заранее. Кластер одноузловой — метки/тейнты вешаем на
единственную ноду. Имя ноды: `kubectl get nodes`.

> **Порядок:** задание 6 (taint) выполняйте **последним** — тейнт `NoSchedule`
> помешает планировать поды без toleration.

```bash
NODE=$(kubectl get nodes -o jsonpath='{.items[0].metadata.name}')
```

## 1. Метка на ноду

```bash
kubectl label node $NODE disk=ssd
```

## 2. Pod с nodeSelector

```bash
kubectl -n sched run sel --image=nginx --overrides='
{"spec":{"nodeSelector":{"disk":"ssd"}}}'
```

## 3. Pod с nodeAffinity

```bash
cat <<EOF | kubectl apply -f -
apiVersion: v1
kind: Pod
metadata: {name: aff, namespace: sched}
spec:
  affinity:
    nodeAffinity:
      requiredDuringSchedulingIgnoredDuringExecution:
        nodeSelectorTerms:
        - matchExpressions:
          - {key: disk, operator: In, values: [ssd]}
  containers:
  - {name: c, image: nginx}
EOF
```

## 4. Pod с requests/limits

```bash
cat <<EOF | kubectl apply -f -
apiVersion: v1
kind: Pod
metadata: {name: res, namespace: sched}
spec:
  containers:
  - name: c
    image: nginx
    resources:
      requests: {cpu: "100m", memory: "64Mi"}
      limits:   {cpu: "200m", memory: "128Mi"}
EOF
```

## 5. PriorityClass + Pod

```bash
kubectl create priorityclass high-prio --value=1000

cat <<EOF | kubectl apply -f -
apiVersion: v1
kind: Pod
metadata: {name: prio, namespace: sched}
spec:
  priorityClassName: high-prio
  containers:
  - {name: c, image: nginx}
EOF
```

## 6. Taint + Pod с toleration (делать последним)

```bash
kubectl taint node $NODE dedicated=special:NoSchedule

cat <<EOF | kubectl apply -f -
apiVersion: v1
kind: Pod
metadata: {name: tol, namespace: sched}
spec:
  tolerations:
  - {key: dedicated, operator: Equal, value: special, effect: NoSchedule}
  containers:
  - {name: c, image: nginx}
EOF
```

Документация: [Assigning Pods to Nodes](https://kubernetes.io/docs/concepts/scheduling-eviction/assign-pod-node/),
[Taints and Tolerations](https://kubernetes.io/docs/concepts/scheduling-eviction/taint-and-toleration/),
[Pod Priority and Preemption](https://kubernetes.io/docs/concepts/scheduling-eviction/pod-priority-preemption/).

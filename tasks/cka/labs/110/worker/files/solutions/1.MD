## Задание 2. Ingress для `/cat`

```bash
kubectl create namespace cat
kubectl -n cat create deployment cat --image=viktoruj/ping_pong:latest
kubectl -n cat expose deployment cat --name=cat --port=80 --target-port=8080
```

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: cat
  namespace: cat
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
spec:
  ingressClassName: nginx
  rules:
  - http:
      paths:
      - path: /cat
        pathType: Prefix
        backend:
          service:
            name: cat
            port: {number: 80}
```

```bash
kubectl apply -f ingress.yaml
curl cka.local:30102/cat    # или IP_ноды:30102/cat
```

## Задание 3. NetworkPolicy в `prod-db`

```yaml
# default-deny ingress
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata: {name: default-deny, namespace: prod-db}
spec:
  podSelector: {}
  policyTypes: [Ingress]
---
# allow from namespace prod
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata: {name: allow-from-prod, namespace: prod-db}
spec:
  podSelector: {}
  policyTypes: [Ingress]
  ingress:
  - from:
    - namespaceSelector:
        matchLabels: {role: prod}
```

```bash
kubectl apply -f netpol.yaml
kubectl get netpol -n prod-db
```

## Задание 4. Миграция Ingress → Gateway API

Исходный `Ingress shop-ingress` (ns `gw`): host `shop.local`, path `/api` → service
`shop:8080`, rewrite `/`. Переносим на пару `Gateway` + `HTTPRoute`.

```bash
# посмотреть, что мигрируем
kubectl get ingress shop-ingress -n gw -o yaml
# доступные GatewayClass (от NGINX Gateway Fabric)
kubectl get gatewayclass
```

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: shop-gw
  namespace: gw
spec:
  gatewayClassName: nginx        # имя из kubectl get gatewayclass
  listeners:
  - name: http
    protocol: HTTP
    port: 80
    hostname: "shop.local"
---
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: shop-route
  namespace: gw
spec:
  parentRefs:
  - name: shop-gw
  hostnames: ["shop.local"]
  rules:
  - matches:
    - path:
        type: PathPrefix
        value: /api
    filters:
    - type: URLRewrite
      urlRewrite:
        path:
          type: ReplacePrefixMatch
          replacePrefixMatch: /       # = rewrite-target: /
    backendRefs:
    - name: shop
      port: 8080
```

```bash
kubectl apply -f gwapi.yaml
kubectl get gateway,httproute -n gw
# после паритет-проверки старый Ingress можно удалить:
# kubectl delete ingress shop-ingress -n gw
```

Соответствие полей: `host` → `hostnames`, `path`+`pathType` → `matches.path`,
`backend.service` → `backendRefs`, `rewrite-target` → `filters: URLRewrite`. Автоматически
черновик можно получить утилитой `ingress2gateway print --providers ingress-nginx -A`.

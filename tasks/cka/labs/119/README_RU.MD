# Lab 119 — Дриллы на скорость и JSONPath

## Описание

Тренажёр скорости для CKA/CKAD: короткие однотипные задания на императивное создание
объектов (Pod, Deployment, Service, ConfigMap, Secret) и отдельный блок на `-o jsonpath` —
извлечение данных из объектов кластера в файлы.

Задания оформлены в экзаменационном стиле с автоматической проверкой командой
`check_result`. Цель дрилла — не разбор теории, а скорость и мышечная память: «набить
руку» на императивных командах и уложиться в темп экзамена (~6-8 минут на задачу).
Файлы-ответы для JSONPath-заданий сохраняются в `/home/ubuntu/answers/`.

## Цель

Закрепить материал глав курса:

- [Глава 3. Работа с kubectl: императивный и декларативный подходы](../../course/03/ru.md) — быстрые императивные команды, `--dry-run=client -o yaml`, генераторы объектов
- [Глава 47. Экзамен CKAD: JSONPath и продуктивность kubectl](../../course/47/ru.md) — извлечение данных через `-o jsonpath`, custom-columns, `--sort-by`
- [Глава 48. Экзамен CKA: тайм-менеджмент и стратегия](../../course/48/ru.md) — распределение времени и порядок решения задач

## Что мы делаем и зачем

| Блок | Задачи | Навык |
|------|--------|-------|
| **Дриллы** | namespace, Deployment, Service, ConfigMap, Secret | скорость императивных команд, `--dry-run` (глава 3) |
| **JSONPath** | версии нод, образы Pod'ов, ClusterIP Service → в файлы | `-o jsonpath`, извлечение данных под таймером (глава 47) |

## Инфраструктура

Окружение разворачивается в AWS (`eu-central-1`) через Terragrunt и состоит из:

| Компонент  | Описание                                                             |
|------------|----------------------------------------------------------------------|
| `vpc`      | VPC `10.10.0.0/16` с публичными подсетями                            |
| `ssh-keys` | SSH-ключи для доступа к нодам                                        |
| `k8s-1`    | Kubernetes `1.35.2` (kubeadm), CNI Calico, metrics-server, одноузловой |
| `worker`   | Рабочая машина с `kubectl` и `check_result`                          |

Инстансы: `t3.medium` (master) Ubuntu `22.04`. Кластер одноузловой — master
«разтейнчен» (снят taint `control-plane`), поэтому поды планируются прямо на него.

## Развёртывание

```bash

```

После создания подключитесь к рабочей машине (worker) по SSH и выполняйте задания

Полезные команды на рабочей машине:

## Задания

---
|        **1**        | **Namespace**                                                |
| :-----------------: | :----------------------------------------------------------- |
| Что делаем          | Создайте namespace (пространство имён) `drill` — в нём будут жить объекты остальных заданий (`kubectl create namespace drill`). |
| Критерии приёмки    | - namespace `drill` существует. |
---
|        **2**        | **Deployment**                                              |
| :-----------------: | :----------------------------------------------------------- |
| Что делаем          | В namespace `drill` создайте Deployment `api` с образом `viktoruj/ping_pong:latest` и 3 репликами (`kubectl -n drill create deployment api --image=viktoruj/ping_pong:latest --replicas=3`). |
| Критерии приёмки    | - Deployment `api` в `drill`: 3 готовые реплики. |
---
|        **3**        | **Service**                                                 |
| :-----------------: | :----------------------------------------------------------- |
| Что делаем          | Опубликуйте Deployment `api` как Service (сервис) типа ClusterIP с именем `api-svc` на порту 80 (`kubectl -n drill expose deployment api --name=api-svc --port=80 --target-port=8080`). |
| Критерии приёмки    | - Service `api-svc` (ns `drill`): Endpoints не пусты. |
---
|        **4**        | **ConfigMap и Secret**                                      |
| :-----------------: | :----------------------------------------------------------- |
| Что делаем          | В namespace `drill` создайте ConfigMap `cfg` с ключом `APP_COLOR=blue` и Secret `sec` с ключом `PASSWORD` (значение любое). Используйте `--from-literal`. |
| Критерии приёмки    | - `cfg.data.APP_COLOR=blue`;<br/>- у Secret `sec` есть ключ `PASSWORD`. |
---
|        **5**        | **JSONPath: версии нод**                                    |
| :-----------------: | :----------------------------------------------------------- |
| Что делаем          | Извлеките `kubeletVersion` всех нод через `-o jsonpath` и сохраните результат в файл `/home/ubuntu/answers/versions.txt` (путь `{.items[*].status.nodeInfo.kubeletVersion}`). |
| Критерии приёмки    | - Содержимое `versions.txt` совпадает с выводом `jsonpath` по нодам. |
---
|        **6**        | **JSONPath: образы Pod'ов**                                 |
| :-----------------: | :----------------------------------------------------------- |
| Что делаем          | Извлеките образы контейнеров всех Pod'ов namespace `drill` через `-o jsonpath` и сохраните в `/home/ubuntu/answers/images.txt` (путь `{.items[*].spec.containers[*].image}`). |
| Критерии приёмки    | - Набор образов в `images.txt` совпадает с выводом `jsonpath` по Pod'ам `drill`. |
---
|        **7**        | **JSONPath: ClusterIP Service**                             |
| :-----------------: | :----------------------------------------------------------- |
| Что делаем          | Извлеките ClusterIP Service `api-svc` через `-o jsonpath` и сохраните в `/home/ubuntu/answers/api-ip.txt` (путь `{.spec.clusterIP}`). |
| Критерии приёмки    | - Значение в `api-ip.txt` совпадает с `.spec.clusterIP` Service `api-svc`. |
---

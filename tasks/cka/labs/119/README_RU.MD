# Lab 119 — Дриллы на скорость и JSONPath

## Описание

Тренажёр скорости для CKA/CKAD: короткие однотипные задания на императивное создание
объектов (под, деплой, сервис, ConfigMap, Secret) и отдельный блок на `-o jsonpath` —
извлечение данных из объектов кластера в файлы. Цель — «набить руку» и уложиться в
темп экзамена (~6-8 минут на задачу), а не разбирать теорию.

Файлы-ответы для JSONPath-заданий сохраняются в `/home/ubuntu/answers/`.

## Цель

Закрепить главы курса:

- [Глава 3. Работа с kubectl: императивный и декларативный подходы](../../course/03/ru.md)
- [Глава 47. Экзамен CKAD: JSONPath и продуктивность kubectl](../../course/47/ru.md)
- [Глава 48. Экзамен CKA: тайм-менеджмент и стратегия](../../course/48/ru.md)

## Что мы делаем и зачем

| Блок | Задачи | Навык |
|------|--------|-------|
| **Дриллы** | ns, deployment, service, configmap, secret | скорость императивных команд, `--dry-run` |
| **JSONPath** | версии нод, образы подов, ClusterIP сервиса → в файлы | `-o jsonpath`, извлечение данных под таймером |

## Задания

---
|        **1**        | **Namespace**                                                |
| :-----------------: | :----------------------------------------------------------- |
| Что делаем          | Создать неймспейс `drill` |
| Критерии приёмки    | - Неймспейс `drill` существует |
---
|        **2**        | **Deployment**                                              |
| :-----------------: | :----------------------------------------------------------- |
| Что делаем          | В `drill` создать deployment `api` (образ `viktoruj/ping_pong:latest`, 3 реплики) |
| Критерии приёмки    | - `api`: 3 готовые реплики |
---
|        **3**        | **Service**                                                 |
| :-----------------: | :----------------------------------------------------------- |
| Что делаем          | Открыть `api` как ClusterIP-сервис `api-svc` на порту 80 |
| Критерии приёмки    | - `api-svc` (ns `drill`): Endpoints не пусты |
---
|        **4**        | **ConfigMap и Secret**                                      |
| :-----------------: | :----------------------------------------------------------- |
| Что делаем          | В `drill`: ConfigMap `cfg` (`APP_COLOR=blue`) и Secret `sec` (`PASSWORD=...`) |
| Критерии приёмки    | - `cfg.data.APP_COLOR=blue`<br>- у `sec` есть ключ `PASSWORD` |
---
|        **5**        | **JSONPath: версии нод**                                    |
| :-----------------: | :----------------------------------------------------------- |
| Что делаем          | Сохранить kubeletVersion всех нод в `/home/ubuntu/answers/versions.txt` |
| Критерии приёмки    | - Содержимое совпадает с `jsonpath` по нодам |
---
|        **6**        | **JSONPath: образы подов**                                  |
| :-----------------: | :----------------------------------------------------------- |
| Что делаем          | Сохранить образы контейнеров всех подов ns `drill` в `/home/ubuntu/answers/images.txt` |
| Критерии приёмки    | - Набор образов совпадает с `jsonpath` по подам `drill` |
---
|        **7**        | **JSONPath: ClusterIP сервиса**                             |
| :-----------------: | :----------------------------------------------------------- |
| Что делаем          | Сохранить ClusterIP сервиса `api-svc` в `/home/ubuntu/answers/api-ip.txt` |
| Критерии приёмки    | - Значение совпадает с `.spec.clusterIP` сервиса |
---

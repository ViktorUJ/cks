# Лаба 119 — эталонное решение (дриллы на скорость + JSONPath)

Смысл лабы — набить скорость императивными командами и `--dry-run`, а также натренировать
`-o jsonpath`. Файлы-ответы сохраняем в `/home/ubuntu/answers/`.

Ускорители (добавьте в сессию):

```bash
alias k=kubectl
export do="--dry-run=client -o yaml"
```

## Дриллы

```bash
# 1. namespace
kubectl create namespace drill

# 2. deployment api, 3 реплики
kubectl -n drill create deployment api --image=viktoruj/ping_pong:latest --replicas=3

# 3. service ClusterIP для api на 80 порту
kubectl -n drill expose deployment api --name=api-svc --port=80 --target-port=8080

# 4. configmap и secret
kubectl -n drill create configmap cfg --from-literal=APP_COLOR=blue
kubectl -n drill create secret generic sec --from-literal=PASSWORD=s3cr3t
```

## JSONPath

```bash
mkdir -p /home/ubuntu/answers

# 5. версии kubelet всех нод
kubectl get nodes -o jsonpath='{.items[*].status.nodeInfo.kubeletVersion}' \
  > /home/ubuntu/answers/versions.txt

# 6. образы всех контейнеров подов в неймспейсе drill
kubectl -n drill get pods -o jsonpath='{.items[*].spec.containers[*].image}' \
  > /home/ubuntu/answers/images.txt

# 7. ClusterIP сервиса api-svc
kubectl -n drill get svc api-svc -o jsonpath='{.spec.clusterIP}' \
  > /home/ubuntu/answers/api-ip.txt
```

Полезные JSONPath-приёмы для экзамена:

```bash
# отсортированный вывод
kubectl get pods -A --sort-by=.metadata.creationTimestamp
# custom-columns
kubectl get pods -n drill -o custom-columns=NAME:.metadata.name,IMAGE:.spec.containers[*].image
# range в jsonpath
kubectl get nodes -o jsonpath='{range .items[*]}{.metadata.name}{"\t"}{.status.nodeInfo.kubeletVersion}{"\n"}{end}'
```

Документация: [JSONPath Support](https://kubernetes.io/docs/reference/kubectl/jsonpath/),
[kubectl Quick Reference](https://kubernetes.io/docs/reference/kubectl/quick-reference/).
